I was mounting my RFS via NFS on my target board but was facing this problem
 
Looking up port of RPC 100003/2 on 10.3.2.1
Looking up port of RPC 100005/1 on 10.3.2.1
VFS: Mounted root (NFS filesystem).
Freeing unused kernel memory: 40k freed
Warning: unable to open an initial console.
Kernel panic: No init found.  Try passing init= option to kernel.

I was able to correct this problem myself.  I did a google search on the title string above 
and found that many people have this problem.  The best clue came from this link Udev howto



It turns out that udev is nearly automatic for all devices accept console and null. Apparently it is necessary to include the traditional links in /dev/ for these devices as follows:




1) Goto NFS server
2) # cd /nfsboot/FS
3) # cd dev/
4) # mknod -m 660 console c 5 1
5) # mknod -m 660 null c 1 3
6) reboot target and try nfs boot now
