cross compiling:

$ CC=arm-linux-gnueabi-gcc ./configure --prefix=/media/sf_work/curl-7.20.0/build --host=arm-linux-gnueabi  --with-random=/dev/urandom --with-sysroot=/usr/local/arm-linux-gnueabi/sys-root/  --enable-debug --disable-curldebug

$ make 

$ make install

this will compile curl libraries with debug symbols.

so you can place them on target and use gdb to set break points and debug symbols.

just replace the old libcurl libraries with the new one.

NOTE: curl options are best to configure curl . CURL_OPT_* must go through them whenever a problem occurs.


CURL DEBUGGING:
curl_easy_setopt(curl, CURLOPT_VERBOSE, 1L);
FILE * filep = fopen("/home/root/curl_dump", "a");
curl_easy_setopt(curl, CURLOPT_STDERR, filep);

this will create file curl_dump with all verbose curl messages.

you can place your own debug messages in curl as well using finfo()...something like that
