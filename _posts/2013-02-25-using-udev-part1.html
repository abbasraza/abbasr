I was facing a problem that on target whenever i mounted the mmc card and then at sometime ejected the card, the mount point was not getting removed automatically. For this i tried to add settings to /etc/fstab but it was of no use. Although study says that adding device settings to /etc/fstab will mount that device automatically on boot. I don't know the reason right now ??

Then i looked into udev rules to solve my issue.

1) What was there for mmc in udev rules

root@mx6q:~# grep -Irn mmc /lib/udev/rules.d/
/lib/udev/rules.d/60-persistent-storage.rules:47:KERNEL=="mmcblk[0-9]", SUBSYSTEMS=="mmc", ATTRS{name}=="?*", ATTRS{serial}=="?*", ENV{ID_NAME}="$attr{name}", ENV{ID_SERIAL}="$attr{serial}", SYMLINK+="disk/by-id/mmc-$env{ID_NAME}_$env{ID_SERIAL}"
lib/udev/rules.d/60-persistent-storage.rules:48:KERNEL=="mmcblk[0-9]p[0-9]", ENV{ID_NAME}=="?*", ENV{ID_SERIAL}=="?*", SYMLINK+="disk/by-id/mmc-$env{ID_NAME}_$env{ID_SERIAL}-part%n"
root@mx6q:~# 

root@mx6q:~# grep -Irn mmc /etc/udev/rules.d/
root@mx6q:~# 


2) looks like the add/remove actions for mmc are not there and must be added. So create a rule file "11-mmc.rules" which will contains following stuff
root@mx6q:~# cat /etc/udev/rules.d/11-mmc.rules 
ACTION=="add" KERNEL=="mmcblk[0-9]p[0-9]", RUN+="/bin/mount /dev/%k /media/card"
ACTION=="remove" KERNEL=="mmcblk[0-9]p[0-9]", RUN+="/bin/umount /dev/%k"
root@mx6q:~#
